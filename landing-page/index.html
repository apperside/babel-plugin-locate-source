Why You'll Love This Plugin

Precise Location

Instantly find the exact file and line numbers where any component is defined.

Easy Integration

Works with React, Next.js, and any Babel-based project in minutes.

How It Works

1

Add the Plugin

Install the plugin and add it to your Babel configuration.

npm install --save-dev babel-plugin-locate-source
2

Debug in Development

The plugin automatically adds source information to your JSX elements.

// Original JSX
<button className="my-button">Click me</button>

// With location data
<button 
  data-in="Button" 
  data-is="button" 
  data-at="Button.jsx:3" 
  className="my-button">Click me</button>
3

Inspect & Locate

Use your browser's developer tools to inspect elements and find their source location.

No More "Where Is This Component?"

With babel-plugin-locate-source, debugging becomes intuitive. Just inspect the element and immediately see:

  • data-at: The file and line numbers (Card.jsx:14-24)
  • data-is: The component name (Card)
  • data-in: The parent component (HomePage)

Add the included debug styles for visual highlights and tooltips on hover!

Getting Started

// vite.config.js
import react from "@vitejs/plugin-react";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [
    react({
      babel: {
        plugins: [["babel-plugin-locate-source", { enabled: true }]],
      },
    }),
  ],
});
// .babelrc
{
  "presets": ["next/babel"],
  "env": {
    "development": {
      "plugins": [
        "babel-plugin-locate-source"
      ]
    }
  }
}

Note: Using Babel with Next.js will disable the SWC compiler. This is a necessary trade-off for getting component source information.

// babel.config.js
module.exports = {
  plugins: [
    ["babel-plugin-locate-source", 
     { enabled: process.env.NODE_ENV === "development" }]
  ]
};

Enhance Debugging with Visual Styles

// Import styles
import 'babel-plugin-locate-source/debug-styles.css';

// Add toggle function
function toggleDebugMode() {
  document.body.classList.toggle('debug-mode');
}

// Usage in component
<button onClick={toggleDebugMode}>
  Debug Mode
</button>
